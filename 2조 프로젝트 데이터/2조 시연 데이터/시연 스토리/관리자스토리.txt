1. 관리자 로그인페이지(/admin/login)
	- CASE1 : 이메일 형식이 아니면 오류 메세지가 발생합니다.
	- CASE2 : 이메일 주소와 비밀번호를 잘 못 입력하면 오류 메세지 발생합니다.
	- CASE3 : DB에 저장된 이메일 주소와 비밀번호를 동일하게 입력 시
		admin main(/admin/main)페이지로 이동합니다. 

2. 관리자 메인페이지(/admin/main)
	- CASE1 : 대시보드 왼쪽 상단에 2022년 월별로 회원가입 수에 따른
		곡선 그래프로 증감 추이를 확인할 수 있습니다.
	- CASE2 : 대시보드 오른쪽 상단에서는 2022년 월별로 이 웹사이트에 
		등록된 병원 가입 수를 막대 그래프로 확인할 수 있습니다.
	- CASE3 : 공공데이터 포털에서 추출한 데이터들을 페이징 처리하여
		조회할 수 있습니다. 왼쪽 숫자 탭을 이용하여 페이지 당
		조회하고 싶은 수를 조절할 수 있고, 오른쪽 검색탭에서 키워드를
		이용해 해당 키워드를 참조하고 있는 데이터들만 조회할 수 있습니다.
		ex) 병원이름, 주소, 전화번호 어떤 키워드든
3. 회원관리페이지(/admin/member, /admin/hospital)
	- CASE1 : 회원측과 병원측 역시 이 웹사이트에 등록된 데이터들을 
		조회할 수 있습니다. 위에 보았던 테이블과 마찬가지로 검색기능과
		페이징을 사용할 수 있습니다.
4. 리뷰 삭제요청(/admin/review)
	- CASE1 : 병원측에서 관리자측으로 삭제 요청을 한 리뷰가 조회됩니다.
		오른쪽에 삭제여부 셀렉트 박스를 이용해서 리뷰 삭제 허가 여부를	
		설정할 수 있습니다. 관리자 측은 삭제 여부가 대기중인 데이터만 
		조회할 수 있고, 삭제 수락, 삭제 반려 시 데이터들은 조회할 수 없고
		반영 즉시 데이터가 업데이트 됩니다.
5. 문의사항(/admin/contactad)
	- CASE1 : 병원측에서 관리자측으로 문의한 내용들이 조회됩니다.
		제목 링크를 클릭하면 문의 사항 내용을 확인할 수 있고,
		답변을 하면 답변 여부가 변경됩니다.
	- CASE2 : 병원에서 문의한 내용의 답변 여부를 병원측에서 확인 할 수
		있고, 내용도 확인할 수 있습니다.
6. 공지사항(/admin/notice)
	- CASE1 : 관리자가 공지 등록을 할 수 있고, 해당 내용을 조회할 수 있습니다.
	- CASE2 : 수정 버튼을 눌러 제목과 내용을 수정할 수도 있습니다.
7. 로그아웃
	- CASE1 : 오른쪽 상단에 사람 모양을 클릭하고 로그아웃을 누르면
		관리자 로그인 페이지로 이동하게됩니다.
